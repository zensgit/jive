# Jive ADB 验证脚本兼容性修复（auto.dev）

日期：2026-02-18

## 背景

在设备包名为 `com.jivemoney.app.auto.dev` 的场景下，原 `scripts/verify_dev_flow.sh` 存在以下问题：

1. 文本匹配过于模糊，`设置` 会误命中 `自动记账设置`。
2. 不同构建/页面结构下，部分断言（如 `图标强制单色`、`修改` 菜单）并不总是存在，脚本会提前失败。
3. 周期规则页点击 `分类` 后，新版分类选择器文案不一定含 `选择分类`，导致误判失败。

## 本次改动

文件：`scripts/verify_dev_flow.sh`

1. 新增精确标题点击能力：
   - `tap_title_once(...)`
   - `tap_title_with_scroll_small(...)`
   - 用于精确匹配 `content-desc="标题&#10;..."`，避免 `设置` 误命中。

2. `设置 -> 分类图标风格` 检查改为可兼容：
   - 存在 `设置` 入口则执行原断言。
   - 不存在时记录日志并跳过该段，不中断后续验证。

3. 分类管理中的强制单色检查改为“存在即校验，不存在则跳过”：
   - 父类新建页
   - 子类新建页
   - 编辑页及后续开关/徽标恢复链路

4. 周期规则里的分类选择器判定增强：
   - 支持 `选择分类` 文案
   - 或包含 ` · ` 的分类项
   - 或存在 `关闭` 语义节点

## 验证命令

```bash
bash -n scripts/verify_dev_flow.sh
bash scripts/verify_dev_flow.sh com.jivemoney.app.auto.dev
```

## 验证结果

- 结果：`PASS`
- 产物目录：`/tmp/jive-verify-20260218-194316`
- 关键覆盖：
  - 首页 -> Debug Sheet
  - 分类管理入口可达
  - 周期记账 -> 新建规则 -> 分类选择器可达
  - 预算管理页加载无持续转圈

## 结论

该脚本已可在 `com.jivemoney.app.auto.dev` 结构下稳定执行，减少因 UI 变体造成的误失败，可继续作为日常真机回归基线。
